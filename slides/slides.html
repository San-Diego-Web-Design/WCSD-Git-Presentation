What is it?

This look familiar?

— "Hey Joe, I uploaded the file, did you get it?"

— "Uh oh, I think I overwrote it!"

You want to avoid exactly that.

Enter version control!

Version control makes it easier for you and your colleagues to work with files, share them and keep track of versions, without ever overwriting each other.

Version Control Systems (VCS)

  • SVN (what WordPress.org uses)
  • Git
  • Others... (cvs, hg)

Why do it?

Simple. It makes your life as a developer easier.

Why do it?

  • All your work is versioned
  • Organize your work better, stash and come back to it later
  • Seamless collaboration
  • Separate production from development
  • Separate bug fixes from feature development
  • Easier to roll-back

Subversion (SVN)

SVN relies on a single "central repository" of the code. It requires a network connection to access and in most cases if the repository crashes, you're in trouble.

  • core.svn.wordpress.org
  • plugins.svn.wordpress.org
  • themes.svn.wordpress.org

Git

Every Git clone is a full-fledged repository with complete history and full revision tracking capabilities, not dependent on network access or a central server

— git-scm.com

How to do it?

At it's root Git is a command-line tool

Even if you choose to use a GUI tool for your Git workflow, you should know how it works behind the scenes

Some basic commands

$ git init
creates a new git repository

$ git clone [url]
clones an existing repository

$ git add [files]
stages files to be committed

$ git commit -m "your message"
commits your files to the repo (always write meaningful commit messages)

More basic commands

$ git status
compare working directory vs. staging area

$ git fetch
download repository data from a remote location

$ git pull
fetch repository and attempt to merge into the current branch

$ git push
push repository to remote location

A few more advanced commands

$ git branch [branchname]
create a new branch

$ git merge
merge two branches together

$ git tag
create a snapshot of your repository

$ git stash
stash your changes and come back later

There's a few others. Find full details at: gitref.org

Do you build client sites?

You probably want a private repository.

You probably want to automate code deployment.

You definitely want to version your work.

Private Repositories

Host a private Git repository containing the entire WordPress installation

Ignore the .htaccess and wp-config.php file from the repository, or setup a local-config.php file.

Local config set-up

Include your wp-config.php in the repo, remove the database connection information and add the following to it:

include_once('local-config.php');

Then, setup local-config.php (gitignored) for each environment (dev, staging, production) with local database info + WP_DEBUG

Where to host private repos

  • Github private repositories — What I use for projects
  • Beanstalk — I also use it
  • BitBucket — No personal experience with it
  • Assembla — They have a free plan
  • Unfuddle — Great for more complete project management
  • Roll your own (gitolite, gerrit, others ) — If you're more hardcore

Deploying sites

You want to develop locally and automatically push code (usually a branch form your repo) up to your staging and production envrionments

  • Beanstalk — Integrates repository hosting & deployments
  • DeployHQ — Integrates deployments with GitHub
  • Simple cron script on your server — $ git pull
  • Capistrano — Requires Ruby
